## 四、文档要求与实际代码的差异点

### 4.1 技术栈差异

| 文档设想 | 实际实现 |
|----------|---------|
| Rust 桌面端（Tauri） | React + TypeScript + Vite + Tauri 前端 |
| `src/novel/mod.rs` Rust 模块 | `src/lib/novel/*.ts` TypeScript 模块 |
| Rust 后端逻辑 | 全部在前端 TypeScript 实现 |

**说明**：文档中设想的 Rust 后端模块（`mod.rs`、`domain.rs` 等）实际以 TypeScript 实现在前端。这不影响功能完整性，只是实现方式不同。核心逻辑（上下文引擎、章节摄取、审稿）全部通过 LLM API 调用完成，不需要 Rust 后端。

### 4.2 存储方式差异

| 文档设想 | 实际实现 |
|----------|---------|
| `.novel/` 旁路存储 | ✅ 已实现 `.novel/snapshots/`、`.novel/cognition-state.json` 等 |
| 原 wiki 结构复用 | ✅ 章节/大纲存于 `wiki/` 目录 |
| 实体页面存于 `wiki/entities/` | ✅ writeSnapshotToWiki 实现 |

### 4.3 未实现的文档设想

| 文档设想 | 状态 | 原因 |
|----------|------|------|
